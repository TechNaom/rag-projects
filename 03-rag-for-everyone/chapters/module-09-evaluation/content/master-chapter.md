# Module 9: Evaluation — Master Chapter

    **Outcome:** Measure retrieval quality, answer faithfulness, latency, and cost using practical metrics and frameworks.

    ## Why this module matters

    Without evaluation, every demo feels great. With evaluation, you finally learn whether the system is actually good enough to trust.

    ## Subtopics

    - Recall@K
- Precision@K
- MRR
- NDCG
- Faithfulness
- Context Precision
- Answer Relevancy
- Groundedness
- Latency
- Cost
- Evaluation frameworks: Ragas and DeepEval

    ## Example-driven lens

    Use one running scenario through the entire module. A policy assistant, support copilot, or internal developer helper works well because it makes the retrieval trade-offs concrete.

    ## What a strong learner should be able to do

    - Explain the topic in plain language
    - Show one example
    - Name one trade-off
    - Defend one design choice
