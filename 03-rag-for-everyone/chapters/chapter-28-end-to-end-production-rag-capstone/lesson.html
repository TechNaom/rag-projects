End-to-End Production RAG Capstone

The capstone turns the course into a portfolio-grade product. The goal is not a toy chatbot. The goal is a system people revisit because it teaches architecture, evaluation, operations, and product judgment.

Three production-grade products with high revisit value

  • Policy Intelligence Desk: enterprise policy assistant with access control, citations, conflict handling, and audit logs.
  • Incident Runbook Copilot: SRE assistant that retrieves runbooks, ranks by service/severity, proposes steps, and captures feedback.
  • Research Knowledge Studio: document research workspace with source comparison, evidence boards, claim checks, and exportable briefs.

Five Capstone Tracks

C01

Enterprise Policy RAG

Build ingestion, metadata, RBAC, hybrid retrieval, grounded answers, citation audits, and release gates.

C02

SRE Runbook RAG

Build service-aware retrieval, incident severity routing, step-by-step answers, freshness checks, and post-incident evals.

C03

Legal Contract Explorer

Build clause search, conflict detection, authority ranking, refusal behavior, and human-review workflow.

C04

Customer Support Knowledge Hub

Build FAQ plus ticket history retrieval, answer quality scoring, feedback loop, and cost controls.

C05

Research Briefing Studio

Build multi-document evidence boards, citation-rich summaries, claim verification, and exportable brief generation.

Universal Build Roadmap

1. Choose domain and users
2. Collect public-safe corpus
3. Parse and clean documents
4. Design metadata and access model
5. Chunk and index
6. Build retrieval + reranking
7. Inject context with citations
8. Add grounded prompt and refusals
9. Add retrieval and answer evals
10. Add observability and cost tracking
11. Package UI, README, architecture diagram, demo script
12. Write production trade-off report

What Makes It Portfolio Grade

Include architecture diagrams, eval reports, trace examples, failure analysis, deployment plan, security notes, cost model, roadmap, and demo scenarios. A reviewer should see how you think, not only what you coded.

Project Roadmap

Use the capstone planner to choose a track, generate the build backlog, and decide minimum viable product versus production hardening milestones.
