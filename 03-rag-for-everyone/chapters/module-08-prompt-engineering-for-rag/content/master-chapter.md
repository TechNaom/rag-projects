# Module 8: Prompt Engineering for RAG — Master Chapter

**Outcome:** Write grounded prompts that force citation behavior, structure outputs, and reduce hallucinations.

## Why this module matters

Good retrieval can still fail if the prompt invites the model to improvise. Prompt design is the final guardrail between evidence and speculation.

## Subtopics

- Context injection
- System prompts
- Citation prompts
- Grounding
- Structured output
- Guardrails
- Hallucination prevention

## Example-driven lens

Use one running scenario through the entire module. A policy assistant, support copilot, or internal developer helper works well because it makes the retrieval trade-offs concrete.

## What a strong learner should be able to do

- Explain the topic in plain language
- Show one example
- Name one trade-off
- Defend one design choice
