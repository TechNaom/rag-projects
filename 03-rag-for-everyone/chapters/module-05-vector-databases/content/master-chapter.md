# Module 5: Vector Databases — Master Chapter

    **Outcome:** Understand ANN indexing, filtering, distributed design, and practical trade-offs across popular vector databases.

    ## Why this module matters

    Vector search feels magical until the corpus reaches production scale. Then index structures, filtering, and replication start deciding your latency bill.

    ## Subtopics

    - ANN vs Exact Search
- HNSW
- IVF
- PQ
- OPQ
- DiskANN
- ScaNN
- Filtering
- Metadata search
- Hybrid search
- Distributed vector databases
- Sharding
- Replication
- Database comparisons: Pinecone, Weaviate, Qdrant, Milvus, and Chroma

    ## Example-driven lens

    Use one running scenario through the entire module. A policy assistant, support copilot, or internal developer helper works well because it makes the retrieval trade-offs concrete.

    ## What a strong learner should be able to do

    - Explain the topic in plain language
    - Show one example
    - Name one trade-off
    - Defend one design choice
