Chapter 4 Project90-120 min build

Build a RAG Pipeline Trace Simulator

Create a small simulator that takes a user scenario and produces an execution trace with route, filters, sources, gates, answer status, and recommended next action.

Project depthThe simulator should prove whether the pipeline is answer-ready, needs retry, must refuse, or should escalate. This is how production teams turn RAG behavior into inspectable engineering data.
starter.pysolution.py

1 Model pipeline stages

Represent route, filters, retrieval, reranking, context assembly, gates, answer policy, and trace.

2 Add gate logic

Detect stale index, missing source family, denied access, weak retrieval, and high-risk incomplete evidence.

3 Produce decision trace

Return answer status, failed gates, source ids, route reason, and recommended next action.
