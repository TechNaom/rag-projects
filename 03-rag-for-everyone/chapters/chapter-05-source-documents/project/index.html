Build a Source Registry Auditor

Create a small tool that reviews source documents and decides whether each source is approved, quarantined, rejected, or requires review before ingestion.

Project depthThe auditor should include format risk, authority status, access level, owner, freshness, required metadata, ingestion readiness, and the first production risk.
starter.pysolution.py

1 Model source records

Include id, format, authority, owner, access, freshness, source URI, and version.

2 Audit readiness

Approve, quarantine, reject, or require review based on governance rules.

3 Produce audit report

Explain source risk, required metadata, and next action for ingestion.
