Project: Hybrid Query Router

Build a router that chooses dense, sparse, hybrid, or high-risk hybrid retrieval and fuses results with RRF.

starter.pysolution.py

What you will build

  • A rule-based query router.
  • An exact-term detector.
  • A reciprocal-rank-fusion function.
  • A route trace with reason, route, controls, and metric.
  • A production recommendation for when hybrid is justified.
Product-level outcome

Route before you retrieve

Your project should explain the retrieval route clearly: "Hybrid because the query contains AZ-4032-B and key-rotation intent; monitor exact-term recall and route latency."
