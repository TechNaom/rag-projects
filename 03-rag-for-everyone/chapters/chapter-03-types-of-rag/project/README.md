# Chapter 3 Project: RAG Architecture Selector

Build a small architecture-selection tool. Given a product scenario, it recommends
a RAG type, explains why, lists production risks, and suggests evals.

## Requirements

1. Define scenarios.
2. Score risk, corpus complexity, workflow complexity, and source format.
3. Recommend a RAG type.
4. Explain why simpler patterns are or are not enough.
5. Suggest evals and observability signals.
6. Produce a decision record.
