Chapter 3 Project90-120 min build

Build a RAG Architecture Selector

Create a small tool that recommends a RAG type from scenario attributes and produces a production decision record.

Project depthThe decision record should include what the architecture is, example client fit, advantages, disadvantages, production risks, engineering profile, accuracy, consistency, scale, evals, and observability signals.
starter.pysolution.py

1 Define scenario inputs

Risk, corpus complexity, workflow type, source formats, client type, and production constraints.

2 Recommend architecture

Choose naive, advanced, corrective, adaptive, agentic, graph, or multimodal RAG with a clear reason.

3 Produce decision record

Explain what it is, examples, pros, cons, advantages, disadvantages, risks, evals, observability, scale, accuracy, consistency, latency, and cost.
