# Chapter 3 Thinking Journal

## Builder Frame

Architecture selection should start with product shape, not trend names.

## Hypothesis

If learners can justify the RAG type, they will avoid both underbuilding and
overbuilding production systems.

## Prompt To Use With GenAI

> Given this product scenario, recommend the simplest RAG architecture that can
> satisfy the risk, corpus, workflow, and eval requirements. Explain why simpler
> patterns are enough or insufficient.

## Critique Checklist

- Did we start from user risk?
- Did we consider corpus shape?
- Did we justify complexity?
- Did we define evals for the selected pattern?
- Did we name operational risks?
