# Chapter 3 Exercises: Choose the Right RAG Type

These exercises are about architecture judgment. For each scenario, choose the
simplest RAG pattern that is strong enough for the product risk.

## Tasks

1. Classify each scenario by RAG type.
2. Explain why a simpler pattern is or is not enough.
3. Name the extra capability needed.
4. Identify the primary production risk.
5. Choose one eval that would prove the architecture works.
6. Add one refusal or escalation rule.
7. Add one observability requirement.
8. Add one cost or latency concern.
