Practice Bank: Quality Failure Patterns

Learn to name answer failures precisely. The faster you can classify a failure, the faster your team can fix the right layer.

starter.pysolution.pyREADME
P01

Unsupported claim

Metric: faithfulness. Action: inspect prompt grounding, context packaging, and unsupported-term patterns.

P02

Right topic, wrong citation

Metric: citation correctness. Action: check claim-to-source support, not only citation presence.

P03

Faithful but incomplete

Metric: completeness. Action: add expected conditions to rubric and benchmark labels.

P04

False refusal

Metric: refusal correctness. Action: tune uncertainty threshold and answer-when-supported policy.

P05

Missed refusal

Metric: safety and refusal correctness. Action: strengthen insufficient-evidence behavior and escalation rules.

P06

High score, high severity

Metric: release gate severity. Action: block serious high-risk failures even when averages look good.

Drill Format

For each failure case in the code, return labels, severity, first action, and release decision. This is how answer evaluation becomes an engineering workflow instead of a scoring spreadsheet.
