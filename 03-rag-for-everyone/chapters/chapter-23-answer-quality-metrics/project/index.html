Project: Answer Quality Scorer

Build a scorer that grades generated answers before release. This becomes the answer-evaluation brain for prompt, model, retriever, and corpus changes.

starter.pysolution.pyREADME

Production roadmap

  • Define a rubric: faithfulness, relevance, citation support, refusal correctness, completeness, and safety.
  • Implement deterministic checks first: citation ids, expected terms, unsupported terms, refusal behavior, and source support.
  • Produce severity-based gates: pass, review, or block.
  • Summarize failures by domain, risk, and query type.
  • Store score traces by prompt, model, retriever, corpus, and benchmark version.

Product Standard

This project is intentionally not a toy "score equals three booleans" script. It models the first version of an evaluator a production team could extend with LLM-as-judge, human review, and observability.

Explain every score
