Interview Questions: Chunking Failure Analysis

Use these to explain chunking problems with production-level clarity.

Q01

How do you debug a wrong RAG answer?

Start with the trace: question, retrieved chunks, ranks, scores, filters, context, citations, prompt, answer, and index version. Verify whether gold evidence was retrieved before blaming the model.

Q02

What is lost context?

The retrieved chunk contains partial evidence but misses the surrounding definition, exception, warning, or condition needed for a faithful answer.

Q03

How do tiny chunks fail?

They match keywords but lack complete meaning. The model receives fragments instead of a usable thought unit.

Q04

How do giant chunks fail?

They retrieve broadly, consume tokens, weaken citations, and bury exact evidence inside unrelated context.

Q05

When is chunking not the root cause?

When gold evidence exists in the index but ranks too low, filters exclude it, reranking fails, prompt formatting hides it, or the model ignores good context.

Q06

How do you reduce overlap noise?

Lower overlap, deduplicate near-identical chunks after retrieval, rank diverse evidence, and monitor duplicate rate.

Q07

What metrics matter for chunking evals?

Recall@k, gold chunk rank, duplicate rate, stale-hit rate, boundary defect rate, parent-link completeness, citation support, and answer completeness.

Q08

How do you safely release a new chunking strategy?

Version the chunker and index, run golden queries, compare against baseline, inspect regressions, release behind a gate, and keep rollback ready.

Q09

How do you handle stale chunk failures?

Track active status, source version, effective date, and index release id. Filter old chunks and validate cleanup during re-indexing.

Q10

What is the best answer if an interviewer asks the ideal chunk size?

There is no universal size. The right size depends on corpus structure, question type, context budget, retrieval metrics, citation quality, and failure analysis.
