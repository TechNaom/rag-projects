Interview Questions: Grounded System Prompts

Q01

What is a grounded system prompt?

A prompt contract that tells the model how to use evidence, cite claims, refuse unsupported answers, and handle conflicts.

Q02

Why is "be accurate" not enough?

It does not define source of truth, citation behavior, or refusal conditions.

Q03

How should the model treat retrieved text?

As evidence data, not as instructions that can override system rules.

Q04

When should a RAG assistant refuse?

When evidence is missing, insufficient, conflicting without authority, restricted, or outside scope.

Q05

What is citation support?

The cited source directly supports the sentence it is attached to.

Q06

How do you handle conflicting evidence?

Use authority and freshness metadata, disclose conflict, cite both sides, or refuse if unclear.

Q07

How do you test a prompt?

Run golden, missing-context, conflicting-source, bad-citation, and prompt-injection cases.

Q08

Why version prompts?

Prompt changes can alter behavior, so traces need prompt version for debugging and rollback.

Q09

Which metrics matter?

Groundedness, citation support, refusal quality, conflict handling, unsupported claim rate, and prompt-injection pass rate.

Q10

What is the senior answer?

A grounded prompt is production policy for answer behavior and should be versioned, tested, and monitored like code.
