Project: Metadata Schema Validator

Build a validator that decides whether metadata records are safe to index and useful for production retrieval.

starter.pysolution.py

What you will build

  • A metadata schema with required fields and controlled values.
  • A validator for identity, access, authority, lifecycle, and source-position metadata.
  • A readiness decision: publish, review, block, or migrate schema.
  • A report with first risk, next action, and first metric to inspect.

1 Define required metadata

Include source id, chunk id, source URI, authority, access level, region, version, effective date, source position, and schema version.

2 Validate controlled values

Reject unknown authority, access, region, document type, and schema version values.

3 Enforce production rules

Restricted chunks need allowed roles. Retired chunks should not answer current production questions.

4 Generate audit reports

Return indexing decision, first risk, next action, and first operational metric.
