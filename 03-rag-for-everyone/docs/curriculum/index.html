Depth lens

The goal is not just to know what embeddings, chunking, or re-ranking are. The goal is to know when each one changes system behavior in a meaningful way.

Decision lens

The modules are grouped so the learner can reason across a full pipeline: ingestion, representation, retrieval, evaluation, and production safeguards.

Learning architecture

What each stage of the roadmap is trying to unlock.

Stage 1 — Why retrieval exists

Modules 1–3 establish the limits of vanilla LLM answers, how documents become context, and why chunking quality changes downstream performance.

Stage 2 — How relevance is manufactured

Modules 4–8 cover embeddings, vector infrastructure, retrieval strategies, re-ranking, and prompt grounding so learners can explain why one search stack beats another.

Stage 3 — How systems are defended in production

Modules 9–12 shift into measurement, operations, advanced patterns, and project execution so learners can defend quality, cost, and reliability decisions.

Per-module asset bundle

Every module keeps the same structure so depth compounds instead of scattering.
