# RAG for Everyone Quality Audits

Every chapter must include a quality audit before it is considered ready for
review. This is the checkpoint that protects the course standard as we scale.

## Audit Dimensions

Each chapter should be reviewed against:

1. Conversational clarity.
2. Production depth.
3. Real-time adoption usefulness.
4. Architecture and diagram quality.
5. Exercise depth.
6. Practice-bank scenario quality.
7. Interview preparation quality.
8. Project implementation strength.
9. GenAI thought-process layer.
10. Navigation and template consistency.
11. Accessibility and readability.
12. Public artifact readiness.

## Scoring

Use a 1-5 score for each dimension:

- 1: missing
- 2: present but weak
- 3: acceptable
- 4: strong
- 5: role-model quality

Any score below 4 should create a follow-up improvement task.
