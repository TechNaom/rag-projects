🔍 Module 1 — Foundation

Understand why RAG exists and the principles behind every design decision

← Back to Course Home | Module 1

What You'll Learn

This module covers the foundational concepts that justify every architectural decision in a RAG system. You'll understand why RAG exists, why LLMs hallucinate, how context windows constrain systems, when to use different RAG architectures, and how queries flow through the entire system.

⏱️ Time Estimate: 12-16 hours (includes reading, coding, and project)
📊 Difficulty: Beginner-friendly | Prerequisites: Basic Python, LLM understanding

Learning Outcomes

✅ Why RAG Is Needed

Understand knowledge cutoff, hallucinations, and long-tail facts that motivate RAG systems.

✅ Root Causes

Learn why hallucinations happen and why they're fundamentally hard to prevent.

✅ System Constraints

Design systems around context window limits and token accounting.

✅ Architecture Choices

Know when to use naive, advanced, agentic, graph, corrective, or adaptive RAG.

✅ End-to-End Flow

See the complete lifecycle from query to answer and how all pieces fit.

🚀 Module 1 Project

Build a Naive RAG System Over Synthetic Banking Data

You'll ingest a synthetic banking policy corpus and build a complete RAG pipeline:

  • Chunk documents into semantic pieces
  • Generate embeddings for each chunk
  • Store in a vector database (Chroma)
  • Retrieve relevant policies for queries
  • Generate grounded answers with citations
  • Evaluate retrieval and generation quality

Time: 4-6 hours | Language: Python
